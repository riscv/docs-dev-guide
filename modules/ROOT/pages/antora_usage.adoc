= Creating a New Specification and Building with Antora

Antora is a static site generator designed for creating documentation sites from multiple repositories. It uses AsciiDoc files and a structured content model to produce a cohesive site. This guide explains how to create a new specification and build it with Antora.

nav.adoc: The navigation file that defines the structure of the site's navigation menu for your specification.

== Working with Overview pages and groups

If your specification belongs to an existing category of specs, for example, Platform Software, use the antora.yml file to specify the page group for your spec. This will ensure that your spec is grouped with related specs in the navigation and on overview pages. For example, if your spec is a platform software spec, you would set the page group to "platform-software" in your antora.yml file:

asciidoc:
  attributes:
    page-group: platform-software

* `modules/ROOT/pages/index.adoc`: The main page for your spec.
* `modules/ROOT/nav.adoc`: Navigation file for your spec.

== 3. Add Content

Write your specification content in AsciiDoc (`.adoc`) files under `modules/ROOT/pages/`.
Update `nav.adoc` to include links to your pages.

== 4. Add to Antora Playbook

In your Antora playbook (e.g., `antora-playbook.yml`), add your spec as a content source:
+
----
content:
  sources:
    - url: ../riscv-new-spec
      branches: HEAD
      start_path: .
----

== 5. Build the Site

Run Antora from the root of your documentation workspace:
+
----
npx antora antora-playbook.yml
----

The new specification should appear in the generated site.
